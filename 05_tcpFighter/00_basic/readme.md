<h1 align="center">격투 게임</h1>

> [서버만 구현] 640x480 맵 안에서 이동하며 싸우는 격투 게임 프로젝트</br>
클라이언트 및 패킷 프로토콜 출처 - https://www.procademy.co.kr/

- 시연 영상은 차후 업데이트
</br>

## 적용한 기술
- 송/수신 링버퍼 적용
- non-blocking, select 모델 사용

## 설명
- 컨텐츠 로직(이동 및 공격)은 일정시간을 두어 업데이트 타이밍 부여
- 클라이언트에서 먼저 이동 후 서버에 통보하는 형식
	- 최초 이동 또는 방향 변경시 이동시작 패킷 전송
	- 서버에서 이동 좌표 계산 후 이동이 종료되었다는 패킷이 오면 최종 좌표 전송
	- 서버에서 계산한 좌표와 클라이언트의 좌표를 비교하여 클라이언트의 좌표가 일정 범위 이상으로 이동했다면 비정상적인 클라이언트로 간주하여 소켓을 끊음
- 캐릭터 공격시 공격범위 계산 후 데미지 패킷 전송
- 캐릭터 사망시 클라이언트의 소켓을 끊는 형식으로 사망처리
- 비록 프로토콜 형식이 비슷하거나 같지만, 실전처럼 패킷 1개당 프로토콜 구조체를 전부 분리하여 관리하였음
